## Шаг 1 (step_id=9862658, free-answer)

Практическое задание 1. Спроектируйте набор проектных инструкций для реального проекта

В этом задании вы выступаете как инженер, который готовит проект к работе с агентом. Ваша задача не просто написать список пожеланий, а оформить короткие и рабочие проектные инструкции, которые помогают агенту быстрее войти в контекст и не делать типовых ошибок.

Ситуация

Представьте, что команда разрабатывает веб-приложение для внутреннего helpdesk-сервиса. У проекта такая структура:

web-app/
  frontend/
  backend/
  tests/
  docs/
  scripts/
  migrations/

Команда хочет, чтобы агент работал по следующим принципам:

	сначала разбирался в структуре проекта и не менял файлы наугад;
	для API-задач сначала читал docs/api.md, а для UI-задач смотрел код в frontend/;
	не трогал migrations/ без явного запроса;
	после изменений запускал релевантные тесты, если они есть;
	в финальном ответе кратко перечислял изменённые файлы и писал, менялась ли структура проекта.

Что нужно сделать

Подготовьте черновик проектных инструкций для такого проекта. ## Шаг 2 (step_id=9862659, free-answer)

Практическое задание 2. Декомпозируйте большую задачу на управляемые шаги

В этом задании вы потренируетесь разбивать большую и рискованную задачу на шаги, с которыми агенту действительно удобно работать.

Ситуация

Команда делает SaaS-приложение для малого бизнеса. Продуктовый менеджер сформулировал задачу так:

Нужно как можно быстрее добавить в приложение модуль "Командные задачи":
- список задач;
- создание задачи;
- назначение исполнителя;

- комментарии к задаче;

Сейчас в проекте уже есть авторизация, база пользователей и базовый frontend/backend-каркас, но новой сущности задач ещё нет. Если отдать такую постановку агенту целиком, есть высокий риск получить слишком большой, трудно проверяемый результат.

	какой результат или артефакт мы ожидаем после завершения шага.

Дополнительное условие

Хотя бы для двух шагов отдельно поясните, почему вы выбрали именно такую стратегию: по слоям, по вертикальному срезу или по фазам. Можно комбинировать стратегии, если это оправдано.
